§ 01 — Technical Specification

Tactical board state evaluation.

RAO-AGI assesses a model's capacity to interpret symbolic board representations and select the optimal move in a deterministic, perfect-information environment. Unlike standard game benchmarks that require deep-tree search, this set focuses on single-step tactical recognition.

Board configurations are represented as a 6x7 grid of symbolic characters. A identifies the current agent, B identifying the adversary, and . representing null space.

Task validity is confirmed by human baseline tests; each board state contains an unambiguous optimal move identifiable through visual inspection and basic rule application.

  • ✓ Static state-to-decision evaluation
  • ✓ Zero-dependency JSON data structure
  • ✓ Deterministic tactical completions
  • ✓ Multi-directional win and block patterns
  • ✓ Cross-provider evaluation harness
  • ✕ Not a simulation environment
  • ✕ Not a training dataset
  • ✕ Not a test of lookahead depth

§ 02 — Data Format

Standardized JSON schema.

Each task is encapsulated in a single JSON object. Training instances contain a solution attribute, which is withheld in evaluation instances.

The board array is ordered top-to-bottom. Standard gravitational physics apply: chips populate the lowest available index within a target column. Current agent is consistently defined as A.

Example: train_003.json

{
  "id": "train_003",
  "board": [
    ".......",   // row 0 (top)
    ".......",
    ".......",
    "...AB..",
    "..ABB..",
    ".ABBB.."    // row 5 (bottom)
  ],
  "current_player": "A",
  "columns": ["0","1","2","3","4","5","6"],
  "solution": "4"
}

Visual validation of tactical solution at column 4.

§ 03 — Execution Guide

Evaluation Infrastructure

The benchmark includes a Python-based evaluation harness supporting local and cloud-based providers. Submission manifests are generated in JSON format for compatibility with the validation utility.

Local

Ollama

Local inference server support for open-weights models.

python eval/run_eval.py \
  --provider ollama \
  --model llama3.2
Cloud API

OpenAI

Support for GPT-4 series and compatible endpoints.

python eval/run_eval.py \
  --provider openai \
  --model gpt-4o-mini
Low Latency

Groq

Optimized inference for high-throughput evaluation.

python eval/run_eval.py \
  --provider groq \
  --model llama-3.3-70b-versatile
Optimized

Anthropic

Native integration with Claude 3.5 series models.

python eval/run_eval.py \
  --provider anthropic \
  --model claude-haiku-4-5-20251001
// Pipeline integration
$ python eval/run_eval.py --provider ollama | python score.py /dev/stdin

// Chain-of-Thought (CoT) evaluation
$ python eval/run_eval.py --provider groq --prompt cot --output out.json
--provider Target inference backend
--prompt minimal | cot
--split training | evaluation
--tasks Execution task limit
--verbose Enable detailed response logging

§ 04 — Evaluation Metrics

Validation and Scoring.

Submissions are structured as JSON maps of task identifiers to column selections. The validation script compares these selections against expected tactical solutions.

Model performance is quantified via raw accuracy. Expected performance for models demonstrating strong symbolic reasoning is 100% across tactical win/block scenarios. 

{
  "train_001": "5",
  "train_002": "4",
  ...
}
$ python score.py submission.json
 
  train_001 → 5 ✓
  train_004 → 0 ✕ (expected 2)
